# 改善ロードマップ

## フェーズ1 — 安定化（優先度: 高）

### 1. 状態の永続化
- **現状**: `StateStore`がインメモリのみ — プロセス再起動でセッション消失
- **対応**: Redis または SQLite で状態を永続化

### 2. `ClaudeClient.judge` の非同期化
- **現状**: 同期呼び出しのため asyncio イベントループをブロックする可能性
- **対応**: `asyncio.get_event_loop().run_in_executor()` でラップ、または `anthropic.AsyncAnthropic` に移行

### 3. リトライ戦略
- **現状**: APIレート制限・タイムアウト時にそのまま例外が上がる
- **対応**: `tenacity` 等でエクスポネンシャルバックオフを実装

---

## フェーズ2 — 品質向上（優先度: 中）

### 4. 結合テスト
- **現状**: ユニットテストのみ（39件）
- **対応**: Orchestrator + StateStore + ClaudeClient を結線した結合テストを追加（Anthropic API はモック）

### 5. シークレット管理
- **現状**: `.env` ファイルのみ
- **対応**: AWS Secrets Manager / Azure Key Vault / 1Password CLI 等への移行

### 6. 水平スケール対応
- **現状**: `StateStore` がシングルトンのためマルチインスタンス不可
- **対応**: 状態を外部ストア（Redis）に移し、インスタンス間で共有可能にする

---

## フェーズ3 — 運用基盤（優先度: 低）

### 7. ヘルスチェック・メトリクス
- **対応**: `/health` エンドポイント（FastAPI等）、Prometheus メトリクス出力

### 8. 外部ロギング
- **対応**: structlog 導入、Datadog / CloudWatch 等への転送

### 9. グレースフルシャットダウン
- **対応**: `SIGTERM` ハンドラで進行中セッションを保存してから終了

### 10. コンテナ化
- **対応**: Dockerfile + docker-compose 作成、CI/CD パイプライン整備

---

## フェーズ4 — ADHD対策機能

### 11. 水分リマインド
- **背景**: 過集中時に水分補給を忘れやすいため
- **対応**: セッション中に定期的（例: 30分ごと）にDiscordへ水分補給を通知
- **コマンド案**: `/hydration start` / `/hydration stop`
- **実装メモ**: `orchestrator.py` の `_inactivity_loop` に相乗りする形で追加可能

### 12. 服薬管理
- **背景**: 服薬タイミングを忘れやすいため
- **対応**: 服薬時刻を登録し、未服薬の場合にDiscordへリマインドを送信
- **コマンド案**: `/meds log` / `/meds status`
- **実装メモ**: 服薬スケジュールの永続化が必要（フェーズ1前提）

### 13. 睡眠ログ
- **背景**: 睡眠と作業パフォーマンスの相関を記録したい
- **対応**: 就寝・起床時刻を記録し、Claudeが翌日のコンディションを考慮した判断を行う
- **コマンド案**: `/sleep` / `/wakeup`
- **実装メモ**: 状態永続化（フェーズ1）が前提
